THỂ LỆ CAPSTONE HACKATHON — W7

"Ship Production-Ready AI in 48 Hours"

Tuần cuối không có course mới, không có lab mới. Cả tuần là một hackathon: trong 48 tiếng, mỗi nhóm dồn toàn bộ những gì đã học từ W1–W6 để ship một AI SaaS Platform chạy thật, deploy public, trên AWS account cá nhân của các bạn. Đây là bài test cuối — gần nhất với một sprint kỹ sư cloud ngoài đời thật.


ĐỀ BÀI — chọn 1 trong 3 domain
Mọi nhóm xây cùng 1 stack kỹ thuật. Chỉ phần chức năng người dùng là khác.
  A. EduTech — "AI Study Buddy": upload PDF/slide -> AI tóm tắt / tạo flashcard / quiz / chat với nội dung
  B. FinTech — "AI Money Coach": upload sao kê (PDF/CSV) -> AI phân loại giao dịch / phân tích chi tiêu / chatbot ngân sách
  C. ProductivityTech — "AI Document Hub": multi-tenant — upload hợp đồng/tài liệu -> AI search / Q&A / tóm tắt

7 NĂNG LỰC BẮT BUỘC — mọi nhóm phải demo đủ
Nguyên tắc: KHÔNG chỉ định service. Mỗi năng lực là một "miền" — nhóm tự chọn service nào để làm, miễn là giải thích được lý do VÀ service đó đã học trong W1–W6.
  1. User Interface       — truy cập qua public URL (HTTPS)
  2. Application Compute  — backend xử lý request + gọi AI + thao tác dữ liệu
  3. AI / ML Feature      — ít nhất 1 tính năng AI chạy end-to-end (Bedrock InvokeModel hoặc KB+Agent...)
  4. Data Persistence     — lưu & đọc lại state qua các session
  5. Object Storage       — S3 cho file/blob
  6. Network Foundation   — tài nguyên cô lập đúng cách (DB KHÔNG public)
  7. Identity & Access    — IAM least-privilege cho mọi service (BẮT BUỘC). Login người dùng là TÙY CHỌN: IAM-only / Cognito / signed URL.


3 NĂNG LỰC TÙY CHỌN (kéo điểm cao hơn, chấm partial credit — làm tốt 1 cái hơn làm dở 3 cái)
  8.  Full Observability     — CloudWatch dashboard + custom metric + alarm (OK/ALARM) + Log Insights query
  9.  Advanced Cost Insights — 3 ảnh Cost Explorer + phân tích cost theo feature + Cost Anomaly alert
  10. Advanced Security      — Pick ONE area: Encryption (KMS CMK+rotation / KMS managed / app-layer) · Audit (CloudTrail+Config / Security Hub / GuardDuty) · Secrets (Secrets Manager+rotation / Parameter Store) · Network (WAF / Shield / SG strictness / Flow Logs). NO service forced.

CHI PHÍ — HARD CAP $100/nhóm
Account cá nhân = tiền thật. Kỷ luật chi phí được chấm điểm.
  < $30  + teardown sạch + Criterion II ≥ 4.0 + Criterion III ≥ 4.0  -> đủ điều kiện bonus (quality gate — cheap mà yếu vẫn KHÔNG bonus)
  $30–60                   -> kỷ luật chi phí tốt
  $60–100                  -> OK, trong cap
  $100+                    -> VƯỢT HARD CAP -> Tiêu chí IV bị chặn ở mức 3
Mẹo giữ dưới $100: dùng free-tier (t3.micro, db.t3.micro) · chọn cheapest sufficient foundation model SAU KHI so sánh $/1M tokens (Haiku/Sonnet/Llama/Titan/Cohere/Mistral) — upgrade chỉ khi đo lường rõ · RDS single-AZ · tránh NAT Gateway (dùng VPC Endpoint) · xóa resource thừa mỗi tối · theo dõi token Bedrock hằng ngày.
Tham khảo estimate chi tiết 3 reference projects + risk analysis ở W7_cost_estimates.md (region ap-southeast-1).

QUY TẮC QUAN TRỌNG — đọc kỹ
  • SHIP, DON'T POLISH. Chấm kiến trúc + happy-path chạy thật. KHÔNG chấm: code đẹp, UI đẹp, error handling đầy đủ, unit test, mobile responsive. Single-file Lambda + form HTML thô = hợp lệ.
  • KHÔNG service mới. Chỉ dùng service đã học W1–W6. "Đã học rồi thì dùng, chưa học thì tập trung cái đã biết."
  • Demo thật VÀ presentation tốt — cả hai đều cần. URL chết = fail thẳng. URL sống nhưng present lúng túng / không giải thích được service choice = chỉ pass. Cả hai mạnh mới hire-able. Đầu tư rehearse Architecture Walkthrough (14p) trước Demo Day.
  • AI phải là call Bedrock thật từ app, KHÔNG tính demo trên Console playground.
  • TEARDOWN hạn CN 1/6 EOD. Không xóa = tiếp tục bị tính tiền cá nhân. Trách nhiệm của nhóm.
